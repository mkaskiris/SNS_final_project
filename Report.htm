1.    Introduction

Since the beginning of 2020 the world is facing a global pandemic which has taken the world by storm. The virus belongs in the coronavirus family and has been named SARS-Cov-2 or covid-19. The virus is similar to the SARS (severe acute respiratory syndrome) outbreak in China 2002 which was contained relatively effectively to the region with around 8000 infected 800 dead.

Covid-19 on the other hand is transmitted more easily with around 20% of the infected needing hospitalization. This has caused over 120 million confirmed cases worldwide and almost 3 million dead. Even though the death rate of the virus is relatively low the high transmission and hospitalization rate has put a huge strain on health care around the world.

The need for accurate forecasting of cases, hospitalizations, Intensive care unit (ICU) patients etc. is critical for the correct allocation of resources as well as future planning for decision makers.

A variety of forecasting tools exist, including statistical and machine learning models. Statistical models include Vector Autoregressions (VAR), Auto Regressive Integrated Moving Average (ARIMA) and Facebook Prophet, which use statistical methods to predict future values of a time series. Machine learning models focus on neural networks, specifically recurrent neural networks (RNN) as a base for the models for their ability to remember past results in the time series and is specifically designed for such a task.

This paper will focus on machine learning neural networks and specifically an RNN layer with the Long Short-Term memory architecture, which are specialized RNN modules that can learn long-term temporal dependencies. This makes them perfect for modelling time series and forecasting problems. A univariate model was trained to predict the cumulative number of cases worldwide using cumulative case data from previous days.

2.    Data Sources, Pre-Processing and EDA

The dataset used contained 71911 measurements and 59 features, 54 of which were numerical 4 categorical and 1 was the date of the measurement. The dataset included data from 6 continents, 214 countries and contained measurements for 425 dates. The dataset was manually downloaded form [1] and stored locally in a csv file.

From the dataset, data for the country of Cyprus were extracted and three feature columns were kept for training and forecasting

1.       cumulative number of cases

2.       daily new cases

3.       ICU patients

A new column for daily percentage change was created for each of the three features to observe the daily fluctuations in the data. As can be seen from figure.1, apart from some sparse fluctuations in certain times the percentage change seems to be a relatively straight line meaning forecasting shouldnt be very difficult as daily changes rarely vary by a lot.

The MinMaxScaler() function from sklearn.preprocessing was used to normalize the input data before being fed to the network since the data comprises of a large variance of data (from 10s to 1e8). This was done to speed up the learning and convergence of the network as well as improve performance.

The data was split into training, validation, and testing set. The first 60% of the data was used in the training set, the next 20% was used in the validation set and the rest 20% was used as a testing set. This split allowed around 80 days in the testing set to test the degradation of the predictions of our model.

The TimeseriesGenerator method available from keras was used to transform the training and validation set into the appropriate format to be used as an input to the Neural Network.

 

3.    Method and Results

To determine the best parameters and architecture for the neural network an iterative process was employed where one parameter was changed at each iteration. Due to the stochastic nature of the algorithm

3.1  Features used

Multiple univariate models were created and fitted. The features used were

1.       Cumulative total number of cases

2.       Daily new cases

3.       ICU patients.

Each model used a single feature for training and forecasting

4.2 Day Lag

Day lag refers to the number of days prior to the output the network will be provided. In this experiment the tested number of days were {5,10,15}.

4.3 Layers

The number and types of layers in the neural network. Three types of layers were selected for this process. LSTM is the Long short-term memory layer enabling the network to learn long temporal dependencies. Dense layer used to output the results of the network. For every LSTM layer apart from the one before the Dense layer the hyperparameter return_sequences was tuned to True to enable each of the layers of the LSTM to learn from the whole sequence of the previous layer. Turning the hyperparameter to False for all layers or to True for all layers was found to reduce the accuracy of the models.

4.4 Nodes

The number of nodes in each of the LSTM networks. The following architectures were tested, where the number are the nodes in each LSTM layer:

1.       LSTM 256->Dense

2.       LSTM 256->128->Dense

3.       LST 256->128->64->Dense

4.       LSTM 256->128->64->32->16->Dense

5.       LSTM 256->128->64->32->16->8->Dense

4.5 Activation function

Two conditions were tested for activation functions

1.       RelU

2.       No activation function

Using an activation function for every LSTM layer did not increase the accuracy of the models significantly relative to no activation function significantly to justify the use of one

4.6 Optimizer

The Adam optimizer was used since it has been shown to perform well for most applications and without requiring specific hyperparameters, it is relatively computationally efficient and as not as prone to be stuck in local minima as Stochastic Gradient Descent (SDG).

4.7 Loss function

The loss function used to train the models was Mean Squared Error as it is one of the loss functions

4.9 Epochs

The number of epochs required to reach convergence varied across the different time series predicted according to the complexity of the series. Simpler series such as the total cases required less epochs. Care was taken not to increase the epochs too much as to overfit the model. Overfitting can be seen when the training loss decreases but the validation loss increases. To determine the optimal number of epochs network#3 was trained on 5,10 and 15 epochs. Network #3 was chosen since it is the midpoint of complexity of all the networks to be tested.

4.10 Batch size

A Batch size of 1 was used due to the limited samples available to us.

4.11 Predictions

The last n entries (where n is the day lag) of the training set were used as input to the trained model to make a prediction for the next day. The prediction was appended to the input and the first entry was removed. This process continued until a prediction was made for all days in the test set.
